# English Vocabulary SRS Plan — Mochi

## Цель

- **Текущий уровень:** B1
- **Целевой уровень:** C1
- **Примерный объём:** ~4 500–6 000 новых слов
  - B1 → B2: ~2 000–2 500 слов
  - B2 → C1: ~2 500–3 500 слов

## Темп изучения

- **15–20 новых карточек в день** + повторения
- **Ожидаемый срок:** ~12–18 месяцев до C1

## Процесс генерации

### Этап 1: Генерация JSON

Claude генерирует слова пакетами по уровням: сначала весь A2 (600 слов), затем B1 (1200), B2 (1700), C1 (2500). Результат сохраняется в `data/a2.json`, `data/b1.json` и т.д. по формату из `docs/data-format.md`.

За один сеанс реалистично сгенерировать ~50–100 слов с полным заполнением полей. Файлы под git — можно проверять, корректировать и наращивать итеративно.

#### Как запросить генерацию

Попросить Claude сгенерировать очередной пакет слов:
- Указать уровень и подуровень (например, «A2 beginner»)
- Указать количество слов (рекомендуется 50–100 за раз)
- Claude читает `docs/field-guidelines.md` для соблюдения формата полей
- Claude читает уже существующие слова в целевом файле, чтобы не дублировать
- Результат дописывается в соответствующий `data/<level>.json`

#### Порядок генерации

1. A2: beginner (250) → intermediate (200) → advance (150)
2. B1: beginner (500) → intermediate (400) → advance (300)
3. B2: beginner (700) → intermediate (550) → advance (450)
4. C1: beginner (1000) → intermediate (800) → advance (700)

Объёмы по подуровням — в `docs/decks.md`.

### Этап 2: Загрузка в Mochi

Скрипт `scripts/upload.py` читает JSON-файлы из `data/` и создаёт карточки через Mochi API. Целевая колода определяется по полям `level` + `sublevel` (структура колод — в `docs/decks.md`).

```bash
# Проверка (без отправки)
python3 scripts/upload.py --dry-run data/a2.json

# Загрузка
python3 scripts/upload.py data/a2.json

# Загрузка нескольких файлов
python3 scripts/upload.py data/a2.json data/b1.json
```

Скрипт использует `requests` (нужен `pip install requests`). Между запросами пауза 0.3 с для соблюдения rate limit.

### Этап 3: Проверка

После загрузки — проверить карточки в интерфейсе Mochi:
- Корректность отображения markdown (списки, bold)
- Правильность распределения по колодам
- Работу трёх сторон карточки (Recognition → Full Info → Cloze)
